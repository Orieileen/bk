亚马逊多店铺 ERP（基于 SP-API）

通过 SP-API 接口实现商品、库存、订单、物流全链路管理与分析，专为多店铺卖家打造，提升协同效率与决策能力。

快速开始 价格说明

能力概览

核心功能
  • 商品管理：批量同步，跨店批量编辑价格/库存预警；监控上架与审核，异常提醒。
  • 库存管理：整合各店铺/仓库库存，低库存预警；周转率分析，补货建议；监控 FBA 状态与费用。
  • 订单管理：统一展示各状态订单，批量发货/退款；追踪状态变更并触发对应流程。
  • 物流配送：同步订单物流，实时监控；可批量生成标签（需授权）；评估物流商服务质量并预警异常。
  • 数据分析：销售排行、库存健康度、履约率、物流成本等报表；按店铺/时间筛选，支持导出。
安全与合规保障
  • 传输安全：TLS 1.2+；静态存储加密：AES-256。
  • 权限最小化：仅申请商品、库存、订单、物流相关必要权限，不收集无关数据。
  • 多角色权限：运营员/管理员等角色分级权限，防止误操作与数据泄露，符合亚马逊数据合规要求。

快速开始 · SP-API 授权

  1. 使用具备管理员权限的亚马逊专业销售账户登录。
  2. 进入卖家中心/开发者中心，为本应用授予权限（商品、库存、订单、物流）。
  3. 完成授权后，在系统绑定您的各个店铺；首次同步将拉取基础数据。
  4. 分配团队角色与权限，配置库存预警阈值与通知方式。
  5. 开始使用商品/库存/订单/物流管理与报表分析功能。

授权范围应与实际业务最小匹配；可随时撤销或调整授权。

使用限制（客户候选资格）

店铺与站点
  • 仅支持亚马逊专业销售账户（Professional Seller Account）。
  • 适配美国、加拿大、墨西哥、欧洲、日本等主流站点。
  • 中东/印度等小众站点需单独适配后使用。
规模与权限
  • 建议月 GMV ≥ 1 万美元，或累计 ASIN 数量 ≥ 50。
  • 使用账号需具备店铺管理员权限，可授权 API 访问与配置。
  • 须遵守 SP-API 使用政策与各站点数据合规要求。

技术使用要求

浏览器
  • Chrome ≥ 90、Firefox ≥ 88、Edge ≥ 90、Safari ≥ 14。
  • 不支持 IE 及版本过低的第三方浏览器。
网络与设备
  • 稳定网络，建议带宽 ≥ 10Mbps，支持 HTTPS。
  • Windows 10+/macOS 11+ 电脑设备（暂不支持移动端直接操作）。
  • 自研系统需支持 RESTful API，对接并具备存储与可视化能力。
